Week 1
IDEA PLANNING
We all share interests in AI - Machine Learning. Drawing inspiration from powerful assistants like J.A.R.V.I.S from "Iron Man", we started our project. One aspect that all group members agreed early in the development process was that the program will be able to record user-submitted macros - a series of recorded actions. This feature will allow users to command the program much more quickly and conveniently.
Week 2
BASIC COMMAND
The most basic commands include moving the mouse cursor, pressing keys, and combining hotkeys. We added more functionalities to smoothen the user experience, such as minimizing and maximizing a window, or launching the default web browser. Unlike the user, the program could not clearly see a button (for example, the minimize window button) and manually click on it.
Week 3
TESTING
Development of Tony relies heavily on testing sessions. Through testing, we added features such as adjusting system volume.By testing, we made the decision that the program would only respond and execute commands when invoked, by saying “Tony” before issuing commands, allowing users to carry out conversations, watch videos, or listen to music.
Week 4
PROJECT LAUNCH
By week 10, our group has finished coding the most essential parts of the program. The voice recognition program runs smoothly, recognizes commands, and quickly executes them; macros are correctly recorded, named, and replayed.
